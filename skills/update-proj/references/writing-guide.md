# 프로젝트 업데이트 작성 가이드

## 말머리 변환 규칙

이슈 제목에서 말머리를 추출하여 결과물 중심 제목으로 변환:

| 이슈 제목 패턴 | 말머리 |
|---------------|--------|
| [파트너스 관리] ... | [파트너스 시스템] |
| 채팅방 ... | [채팅방 운영] |
| 게시판 ... | [게시판] |
| 스터디 ... | [스터디 운영] |
| 뉴스레터 ... | [뉴스레터] |
| LMS ... | [LMS] |
| 대시보드 ... | [대시보드] |
| 자동화 ... | [자동화] |

말머리가 명확하지 않으면 이슈 내용을 분석해서 적절한 말머리 생성

## 이모지 가이드

| 결과물 유형 | 이모지 |
|------------|--------|
| 체계/시스템 개편 | 🔧 |
| 문서화/절차 정리 | 📋 |

| 데이터/분석 | 📈 |

## 좋은 예 vs 나쁜 예

### 한 줄 요약
- ❌ "파트너스 자격 검토 기준을 변경하고 에어테이블 테이블을 생성함"
- ✅ "복잡하고 흩어져 있던 파트너 자격 기준을 단순화하고, 관리 시스템을 에어테이블로 일원화"

### 기존 문제
- ❌ "초대이력 관리 부재"
- ✅ "누가 언제 파트너가 됐는지 카톡 대화 뒤져야 알 수 있었음"

### 변경 사항 표
- ❌ 기술적 용어만 나열
- ✅ 누구나 이해할 수 있는 표현 + 핵심 변경은 **굵게** 강조

## 그룹핑 기준

### 같은 그룹으로 묶을 경우
- 동일한 시스템/영역에 대한 작업
- 순차적으로 진행되어야 하는 작업
- 하나의 목표를 위한 여러 단계

### 별도 그룹으로 분리할 경우
- 완전히 다른 영역
- 독립적으로 진행 가능한 작업
- 다른 담당자/팀이 관여하는 작업

## health 상태 선택 가이드

| 상태 | 선택 기준 |
|------|----------|
| 🟢 onTrack | 계획대로 진행 중, 큰 이슈 없음 |
| 🟡 atRisk | 일정 지연 가능성, 의존성 이슈, 리소스 부족 |
| 🔴 offTrack | 심각한 지연, 블로커 발생, 목표 변경 필요 |
