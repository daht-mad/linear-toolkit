# 이니셔티브 업데이트 출력 포맷

## 기본 포맷

```markdown
## 📊 이니셔티브 현황

**Health**: 🟢 onTrack / 🟡 atRisk / 🔴 offTrack

---

### ✅ 이번 Cycle 핵심 성과

- [프로젝트명] 성과 요약 (1줄)
- [프로젝트명] 성과 요약 (1줄)

### ⚠️ 주요 이슈/리스크

- [프로젝트명] 이슈 내용 → 대응 방안
- 전체 영향: 설명

### 📋 다음 Cycle 중점

- [프로젝트명] 계획 요약
- [프로젝트명] 계획 요약

---

### 프로젝트별 상세

| 프로젝트 | Health | 핵심 진행 |
|----------|--------|-----------|
| 프로젝트A | 🟢 | 한줄 요약 |
| 프로젝트B | 🟡 | 한줄 요약 |
```

## 실제 예시

### 예시 1: 정상 진행 중인 이니셔티브

```markdown
## 📊 이니셔티브 현황

**Health**: 🟢 onTrack

---

### ✅ 이번 Cycle 핵심 성과

- [결제 API 현대화] 신규 결제 수단 연동 완료 (카카오페이, 네이버페이)
- [정산 시스템] 실시간 정산 대시보드 베타 오픈

### ⚠️ 주요 이슈/리스크

- 현재 블로커 없음

### 📋 다음 Cycle 중점

- [결제 API] 토스페이 연동 착수
- [정산 시스템] 파트너사 파일럿 테스트 진행

---

### 프로젝트별 상세

| 프로젝트 | Health | 핵심 진행 |
|----------|--------|-----------|
| 결제 API 현대화 | 🟢 | 2/3 결제수단 연동 완료, 다음 주 토스 착수 |
| 정산 시스템 리뉴얼 | 🟢 | 대시보드 완성, 파일럿 준비 중 |
| 레거시 마이그레이션 | 🟢 | 데이터 정합성 검증 90% 완료 |
```

### 예시 2: 리스크가 있는 이니셔티브

```markdown
## 📊 이니셔티브 현황

**Health**: 🟡 atRisk

---

### ✅ 이번 Cycle 핵심 성과

- [사용자 인증] OAuth 2.0 마이그레이션 완료
- [권한 관리] 역할 기반 접근 제어(RBAC) 설계 확정

### ⚠️ 주요 이슈/리스크

- [레거시 연동] 기존 시스템 API 호환성 이슈 발견 → 추가 어댑터 개발 필요
  - 예상 영향: 1주 지연, 전체 일정 2월 말 → 3월 첫째 주
- [권한 관리] 시니어 개발자 병가로 리뷰 병목 예상

### 📋 다음 Cycle 중점

- [레거시 연동] 어댑터 개발 집중 → 호환성 테스트
- [권한 관리] 외부 리뷰어 섭외 또는 페어 프로그래밍으로 대응

---

### 프로젝트별 상세

| 프로젝트 | Health | 핵심 진행 |
|----------|--------|-----------|
| 사용자 인증 | 🟢 | OAuth 완료, SSO 연동 착수 예정 |
| 권한 관리 | 🟡 | 설계 완료, 구현 시작 (리뷰 병목 우려) |
| 레거시 연동 | 🟡 | API 호환성 이슈로 추가 작업 필요 |
```

```markdown
  - 대응: 경영진 에스컬레이션 진행 중, 대안 벤더 평가 착수
- [대시보드] 데이터 소스 변경에 따른 재설계 필요

- [전체] 벤더 상황 해결이 최우선
- [데이터 파이프라인] 대안 벤더 기술 검토 및 POC
- [대시보드] 기존 데이터 소스 기반 최소 기능 먼저 개발

---

### 프로젝트별 상세

| 프로젝트 | Health | 핵심 진행 |
|----------|--------|-----------|
| 데이터 파이프라인 | 🔴 | 벤더 이슈로 마이그레이션 긴급 착수 |
| 대시보드 | 🟡 | 디자인 완료, 데이터 소스 의존성 대기 |
| 리포팅 자동화 | 🟡 | 파이프라인 안정화까지 개발 보류 |
```

## 작성 팁

### 분량 가이드
- 전체 업데이트: 200-400자 내외
- 프로젝트당: 1-2줄
- 이슈/리스크: 문제 + 영향 + 대응을 한 문장으로

### Health 선택 기준
1. 먼저 하위 프로젝트 health 자동 종합
2. 실제 상황과 맞지 않으면 수동 조정
3. 조정 시 이유를 리스크 섹션에 명시

### 읽는 사람 관점
- 경영진: 전체 진행률, 일정 리스크
- 유관 팀: 의존성, 협업 필요사항
- 팀원: 맥락 이해, 방향성 확인
